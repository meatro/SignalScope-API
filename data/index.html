Observation / Decode Mode

API help 
POST /api/observe  (form-url-encoded)
Fields:
- mode: none | all | specific
- ids: \"0x280:A_TO_B,0x1A0:B_TO_A\" (only for specific)
This controls frame observation and decode activation.
Example:
mode=specific&ids=0x280:A_TO_B

Use all to decode broadly. Use specific to observe specific frame IDs/directions.

Replay / Send Timing

API help 
POST /api/replay/send  (form-url-encoded)
Fields:
- can_id, dlc, direction, data(16 hex chars)
- repeat, interval_us, start_delay_us, auto_start

POST /api/replay  (form-url-encoded)
Fields:
- action=start|stop
- loop_mode=PLAY_ONCE|LOOP_RAW|LOOP_WITH_COUNTER_CONTINUATION
- start_delay_us

This lets users control send timing directly from API.

Timing controls are API-backed: repeat, interval_us, start_delay_us.

Stage BIT_RANGE Rule

API help 
POST /api/rules/stage  (form-url-encoded)
Fields:
- rule_kind=BIT_RANGE
- can_id, direction, start_bit, length
- replace_value, dynamic, enabled
Rule is staged only; commit required to activate.

Stage RAW_MASK Rule

API help 
POST /api/rules/stage  (form-url-encoded)
Fields:
- rule_kind=RAW_MASK
- can_id, direction, mask, value, enabled
mask/value are 16 hex chars (8 bytes).

DBC (Upload + Folder Autoload)

API help 
POST /api/dbc  (text/plain body)
Body:
- full DBC text

POST /api/dbc/autoload  (form-url-encoded or empty)
Reloads from /dbc folder scan order:
1) /dbc/active.dbc
2) /dbc/default.dbc
3) /dbc/vw_pq.dbc
4) first valid /dbc/*.dbc

Boot already auto-loads /dbc/active.dbc, /dbc/default.dbc, /dbc/vw_pq.dbc, then scans /dbc/*.dbc.
